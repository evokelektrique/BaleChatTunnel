# Code of Conduct

## Our Standards

Be respectful, constructive, and focused on the project. Good participation
includes:

- using welcoming and inclusive language
- respecting different viewpoints and experience levels
- giving and receiving technical feedback professionally
- keeping issues and pull requests focused on actionable project work

Unacceptable behavior includes:

- harassment, insults, threats, or personal attacks
- discriminatory language or conduct
- publishing private information without permission
- repeated off-topic or disruptive comments

## Enforcement

Maintainers may edit, hide, or remove comments, issues, pull requests, or other
contributions that violate this code of conduct. Maintainers may also limit or
block participation when needed to keep the project space productive.

## Reporting

Report conduct concerns through GitHub Issues only if the report can be shared
publicly. If private reporting is needed, contact a project maintainer directly
through an available maintainer contact channel.
